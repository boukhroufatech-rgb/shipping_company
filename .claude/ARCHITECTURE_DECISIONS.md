# 🎯 قرارات المعمارية (Architecture Decision Records - ADR)

## الجلسة الحالية

### ❓ **الأسئلة المفتوحة:**

#### 1. محاذة الأرقام (Number Alignment)
```
السؤال: هل الأرقام مثل "عدد الفواتير" و "عدد الحاويات" يجب:
  A) CENTER (محاذاة للمركز) - لأنها وصفية
  B) RIGHT (محاذاة لليمين) - لأنها أرقام
  C) حسب السياق - قد تختلف

الحالة الحالية: CENTER
```

📍 **رأيك؟** (الرجاء الإجابة)

---

#### 2. شمول الكاتالوجات في Schema Tree
```
السؤال: هل نضيف Schemas لجداول الكاتالوجات؟

الخيارات:
  A) نعم، كل شيء موحد (Inclusive)
  B) لا، اترك الكاتالوجات كما هي (Separated)
  C) هجين - فقط الكاتالوجات المعقدة (Hybrid)

الحالة الحالية: Separated (لا schemas للكاتالوجات)
المقترح: Hybrid أو Inclusive
```

📍 **رأيك؟** (الرجاء الاختيار)

---

#### 3. العملات وموضعها
```
السؤال: كيف نعرض العملات والمبالغ؟

الخيار 1 (الحالي):
  ┌─────────────┐
  │ 1,234.56 DA │  ← محاذة RIGHT
  └─────────────┘

الخيار 2 (بديل):
  ┌──────────┬─────┐
  │ 1,234.56 │ DA  │  ← يسار RIGHT، يمين CENTER
  └──────────┴─────┘

الخيار 3 (آخر):
  ┌─────────────┐
  │ DA 1,234.56 │  ← محاذة LEFT (عكس اللغة)
  └─────────────┘
```

📍 **أيهما تفضل؟** (للقراءة والوضوح)

---

#### 4. Auto-Detection vs Explicit
```
السؤال: هل نترك auto-detection للمحاذات أم نجعل كل شيء explicit؟

الحالي (مختلط):
  ✅ بعضها explicit: "align": "right"
  ⚠️  بعضها auto: بدون "align" (يعتمد على النوع)

المقترح (explicit):
  ✅ كل شيء صريح: كل عمود له "align": "..."

الفوائد:
  ✓ وضوح 100%
  ✓ لا التباس
  ✓ احترافي
  ✓ سهل للقراءة
```

📍 **نعم أم لا؟** (أم تفضل الحالة الحالية؟)

## القرارات المحتملة (تحت الدراسة)

### إذا اخترنا الخيار الهجين (Hybrid):

```python
# قاعدة الهجين:

Category 1: Simple Catalogs (بدون Schema)
├─ license_goods
├─ expense_types
├─ account_types
└─ ... (بسيطة جداً)

Category 2: Complex Catalogs (مع Schema)
├─ currencies (لها rates)
├─ suppliers (لها balances)
└─ ... (معقدة أو لها معالجات خاصة)

Category 3: Main Tables (إجباري Schema)
├─ treasury_accounts
├─ customers
├─ shipments
└─ ... (كل البيانات الرئيسية)
```

## القواعس الموحدة المقترحة

### إذا اخترنا Explicit Alignments:

```python
# القاعدة الموحدة:

"amount"      → "align": "right"    # المبالغ المالية
"percentage"  → "align": "right"    # النسب المئوية
"number"      → "align": "right"    # الأرقام بشكل عام (لاحقاً تقرير حسب السياق)
"date"        → "align": "center"   # التواريخ
"status"      → "align": "center"   # الحالات
"text"        → "align": "left"     # النصوص
"currency"    → "align": "center"   # رموز العملات
```

## الخطوات التنفيذية (متعلقة بقراراتك)

### بعد الاتفاق على القرارات:

```
Phase 1: توحيد Schemas
├─ إصلاح كل "amount" بـ "align": "right"
├─ إصلاح جميع auto-detection → explicit
└─ إضافة Schemas للكاتالوجات (إن اخترنا)

Phase 2: تحديث Views
├─ تحديث جميع set_headers() → set_headers_from_schema()
├─ حذف المحاذات اليدوية
└─ اختبار كل جدول

Phase 3: التوثيق
├─ توثيق القواعس الموحدة
├─ إنشاء guide للمطورين الجدد
└─ مراجعة المعمارية النهائية
```

## الملخص: ما تحتاج تقرره

| القرار | الخيارات | التوصية |
|--------|---------|--------|
| محاذة الأرقام | CENTER / RIGHT / السياق | ؟ |
| الكاتالوجات | Inclusive / Separated / Hybrid | Hybrid |
| العملات | مع المبلغ / منفصلة / معكوسة | مع المبلغ |
| Explicit Align | نعم / لا / جزئي | نعم |
