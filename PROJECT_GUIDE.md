## 1. ما هو هذا البرنامج؟

نظام محاسبي متكامل لشركة **استيراد وشحن** تعمل في الجزائر.
الشركة تستورد بضائع من الخارج (الصين/أوروبا) وتوزعها على عملاء محليين.
البرنامج يتتبع كل عملية مالية من لحظة شراء العملة حتى استلام المال من العميل.

## 2. الفاعلون الرئيسيون (Who is Who)

| الفاعل | من هو؟ | أين في البرنامج؟ |
|--------|---------|-----------------|
| **صاحب الرخصة** | شخص يملك رخصة استيراد رسمية، الشركة تستخدمها مقابل عمولة | `modules/licenses` → `CurrencySupplier` (supplier_type=LICENSE) |
| **وكيل الشحن** | شركة شحن تنقل الحاويات من الخارج | `modules/logistics` → `CurrencySupplier` (supplier_type=SHIPPING) |
| **مورد العملة** | شخص يبيع للشركة دولارات/يورو بالسوق الموازية | `modules/currency` → `CurrencySupplier` (supplier_type=CURRENCY) |
| **العميل** | تاجر محلي يطلب بضاعة من الخارج عبر الشركة | `modules/customers` → `Customer` |
| **الشريك** | مساهم في الشركة (apport/retrait/profit) | `modules/partners` → `Partner` |
| **جهة خارجية** | شخص اقترضنا منه أو أقرضناه خارج إطار العمل | `modules/external_debt` → `ExternalContact` |

## 3. مسار العمل الكامل (Workflow)

### المرحلة 1 — تجهيز الأدوات المالية

```
[الخزينة] ← نقطة البداية
    ├── إنشاء الحسابات: CAISSE (صندوق) + COMPTE (بنك) + CCP
    └── كل حساب مرتبط بعملة (DA, USD, EUR...)
```

```
[العملات] ← شراء الدولار من السوق الموازية
    ├── مورد عملة → يبيع X دولار بسعر Y دينار
    ├── الدفع: CASH (يخصم من حساب DA) أو CREDIT (دين للمورد)
    ├── الدولارات تُضاف لحساب USD في الخزينة
    └── إذا CREDIT: يبقى المورد دائناً حتى نسدد له (SupplierPayment)
```

### المرحلة 2 — الاستيراد

```
[اللوجستيك] ← إنشاء عملية استيراد
    │
    ├── خطوة 1: إنشاء الترخيص (ImportLicense)
    │       ├── نختار صاحب الرخصة
    │       ├── نحدد المبلغ الإجمالي بالدولار (total_usd)
    │       └── لا تأثير مالي فوري — فقط تسجيل
    │
    ├── خطوة 2: فتح ملف حاوية (ContainerFile)
    │       ├── رقم الحاوية + رقم الفاتورة
    │       ├── ربط بالترخيص (يأخذ جزءاً من total_usd)
    │       ├── تاريخ الشحن + ميناء التفريغ
    │       └── CBM (حجم البضاعة بالمتر المكعب)
    │
    └── خطوة 3: إضافة المصاريف (ContainerExpense)
            ├── مصاريف الميناء، الجمارك، الناقل...
            └── كل مصروف يُخصم من حساب في الخزينة
```

### المرحلة 3 — توزيع البضائع على العملاء

```
[العملاء] ← بعد وصول الحاوية
    │
    ├── إضافة بضائع العميل (CustomerGoods)
    │       ├── العميل + الحاوية + نوع البضاعة
    │       ├── الحجم بالـ CBM + سعر الـ CBM بالدينار
    │       └── رسوم التعامل = CBM × سعر CBM - الخصم
    │       ← هذا يُنشئ "دين" على العميل تجاه الشركة
    │
    ├── إضافة مصاريف جانبية (CustomerSideCost)
    │       ├── نقل، تفريغ، تخزين...
    │       └── تُضاف لدين العميل
    │
    └── استلام الدفع (CustomerPayment)
            ├── العميل يدفع مبلغاً
            ├── يُضاف للخزينة (حساب DA)
            └── يُنقص من دين العميل
```

### المرحلة 4 — الاستحقاقات والديون

```
[التراخيص] ← تتبع ما يُستحق لصاحب الرخصة
    ├── الدوميسيلياسيون (توثيق البنك)
    ├── الضرائب الجمركية
    ├── الدفعات المُسددة
    └── الرصيد المتبقي (المستحق)
```

```
[الديون الخارجية] ← قروض خارج إطار العمل
    ├── LEND: أقرضنا شخصاً مالاً (يخرج من الخزينة)
    ├── REPAY_LEND: استرجعنا ما أقرضناه (يدخل للخزينة)
    ├── BORROW: اقترضنا من شخص (يدخل للخزينة)
    └── REPAY_BORROW: سددنا ما اقترضناه (يخرج من الخزينة)
```

## 4. مسار الأموال (Money Flow)

```
┌─────────────────────────────────────────────────────────────────┐
│                      FLUX FINANCIER COMPLET                     │
│                    (مسار الأموال الكامل)                        │
└─────────────────────────────────────────────────────────────────┘

                    ┌───────────────────┐
                    │  مورد العملة      │
                    │  (DA → USD)       │
                    └────────┬──────────┘
                             │ يبيعنا دولارات
                             ↓
┌──────────────────────────────────────────────────────────────┐
│                     TRÉSORERIE (الخزينة)                      │
│                                                              │
│  [حساب DA]  ←── يتناقص عند شراء العملة أو دفع مصاريف       │
│  [حساب USD] ←── يتزايد عند شراء العملة                      │
│  [حساب CCP] ←── يُستخدم للتحويلات البنكية                   │
│                                                              │
│  كل دخول وخروج = Transaction (CREDIT أو DEBIT)              │
└──────────────────────────────────────────────────────────────┘
         ↑                                        ↑
         │ يدفع العملاء                           │ يدخل الشركاء أموالاً
    ┌────┴──────────┐                    ┌────────┴──────────┐
    │   العملاء     │                    │    الشركاء        │
    │  (Customers)  │                    │    (Partners)     │
    └────────────── ┘                    └───────────────────┘
         ↑
         │ يستلم بضاعة
    ┌────┴──────────────────────────────────────────────────┐
    │                اللوجستيك                              │
    │                                                      │
    │  ImportLicense → ContainerFile → CustomerGoods       │
    │        ↓                ↓                            │
    │  صاحب الرخصة    مصاريف الحاوية                       │
    │  (يستحق عمولة) (تخرج من الخزينة)                     │
    └──────────────────────────────────────────────────────┘
```

## 5. جدول التأثير المالي لكل عملية

| العملية | أين تحدث؟ | التأثير على الخزينة |
|---------|-----------|---------------------|
| شراء دولارات CASH | Currency | ↓ حساب DA، ↑ حساب USD |
| شراء دولارات CREDIT | Currency | لا تأثير فوري، دين للمورد |
| تسديد مورد العملة | Currency | ↓ حساب DA |
| إنشاء ترخيص | Logistics | لا تأثير |
| مصاريف حاوية | Logistics | ↓ الحساب المختار |
| بضائع عميل | Customers | لا تأثير (تُنشئ دين فقط) |
| مصاريف جانبية للعميل | Customers | لا تأثير (تُنشئ دين فقط) |
| استلام دفع عميل | Customers | ↑ الحساب المختار |
| تحويل بين حسابات | Treasury | ↓ حساب المصدر، ↑ حساب الوجهة |
| إقراض خارجي | External Debt | ↓ الحساب المختار |
| استرجاع قرض خارجي | External Debt | ↑ الحساب المختار |
| اقتراض خارجي | External Debt | ↑ الحساب المختار |
| تسديد دين خارجي | External Debt | ↓ الحساب المختار |
| إيداع شريك | Partners | ↑ الحساب المختار |
| سحب شريك | Partners | ↓ الحساب المختار |

```
Currency (العملة)
    └── ExchangeRate (أسعار الصرف التاريخية)

    └── ExternalTransaction (LEND/BORROW/REPAY...)

    └── PartnerTransaction (CONTRIBUTION/WITHDRAWAL/PROFIT)
```

## 7. كيف يُحسب رصيد العميل؟

```
رصيد العميل (ما يدين به للشركة) =

  رصيد_ابتدائي
+ مجموع(CustomerGoods.total_net)       ← ما عليه من بضاعة
+ مجموع(CustomerSideCost.amount)        ← مصاريف جانبية
- مجموع(CustomerPayment.amount)         ← ما دفعه

إذا النتيجة > 0  → العميل مدين للشركة (له رصيد مستحق)
إذا النتيجة < 0  → الشركة مدينة للعميل (دفع أكثر مما عليه)
```

---

## 8. كيف يُحسب رصيد مورد العملة؟

```
رصيد المورد (ما ندين به له) =

  مجموع(CurrencyPurchase.total_dzd حيث payment_type=CREDIT)
- مجموع(SupplierPayment.amount)

إذا النتيجة > 0  → ندين للمورد (لم نسدد بعد)
إذا النتيجة = 0  → مسوّى بالكامل
```

---

## 9. كيف يُحسب رصيد الخزينة؟

```
رصيد كل حساب = initial_balance + مجموع(CREDIT) - مجموع(DEBIT)

(يُحسب مباشرة من جدول Account.balance الذي يُحدَّث تلقائياً
 عند كل Transaction عبر TreasuryService)
```

## 10. الوحدات وأدوارها الدقيقة

### `modules/treasury/` — الخزينة
- **الدور:** سجل الأموال الفعلية. كل دخول/خروج حقيقي يمر من هنا.
- **القاعدة:** لا تُحرَّك الخزينة مباشرة من أي وحدة — يجب المرور عبر `TreasuryService`.
- **الوظائف الجوهرية:** `create_account`, `transfer`, `add_transaction`.

### `modules/currency/` — العملات
- **الدور:** شراء العملات الأجنبية من السوق الموازية.
- **القاعدة:** تفعيل عملة جديدة → ينشئ حساباً تلقائياً في الخزينة.
- **الوظائف الجوهرية:** `create_purchase`, `pay_supplier`, `toggle_world_currency`.

### `modules/logistics/` — اللوجستيك
- **الدور:** إدارة عمليات الاستيراد من الرخصة حتى وصول الحاوية.
- **المعالج الثلاثي:** ترخيص → حاوية → مصاريف (ShipmentWizard).
- **الوظائف الجوهرية:** `create_license`, `create_container`, `add_expense`.

### `modules/customers/` — العملاء
- **الدور:** تتبع ما يدين به كل عميل للشركة.
- **الوظائف الجوهرية:** `add_goods`, `receive_payment`, `get_customer_balance`.

### `modules/licenses/` — التراخيص
- **الدور:** إدارة الحسابات المالية مع أصحاب الرخص (دوميسيلياسيون، ضرائب، دفعات).
- **يختلف عن اللوجستيك في:** اللوجستيك يُنشئ الرخصة، التراخيص تتابع الأرصدة المالية لصاحبها.

### `modules/external_debt/` — الديون الخارجية
- **الدور:** قروض وديون خارج إطار العمل الأساسي.
- **الأنواع:** LEND (أقرضنا) / BORROW (اقترضنا) + REPAY للتسديد.
- **خاصية:** يدعم متعدد العملات مع تحويل تلقائي للدينار.

### `modules/partners/` — الشركاء
- **الدور:** تتبع أموال المساهمين (إيداع، سحب، توزيع الأرباح).
- **الأنواع:** CONTRIBUTION (إيداع) / WITHDRAWAL (سحب) / PROFIT_ALLOCATION (أرباح).

### `modules/dashboard/` — لوحة التحكم
- **الدور:** ملخص إجمالي لوضع الشركة المالي.
- **يعرض:** أرصدة الخزينة، إجماليات العملاء، ملخص الأنشطة.

### `modules/warehouse/` — المخازن *(قيد التطوير)*
- **الدور:** تتبع مادي للبضائع في المخازن.
- **الربط:** ContainerFile → Warehouse عبر `warehouse_id`.

## 13. categories الخزينة (تمويل الشركاء)

عند تمويل الشركاء (إيداع/سحب)، يتم تسجيل المعاملة في الخزينة dengan:

| Category | الوصف | Direction |
|----------|------|-----------|
| `PARTNER_CONTRIBUTION` | تمويل رأس مال من شريك | CREDIT (دخول) |
| `PARTNER_WITHDRAWAL` | سحب من شريك | DEBIT (خروج) |
| `PARTNER_PROFIT` | تخصيص ربح (كتابي فقط) | لا تأثير على الخزينة |

**Source:** `PARTNER` (+ `source_id` = ID الشريك)
